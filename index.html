Project: Chicken Run

Overview:

This project will be to make a simple third person ‘platforming/deathmatch’ game with approximately five different tutorial levels with each one introducing a new key mechanic that proposes a gameplay challenge for the user and coding challenge for the designers. With an overall goal of making a multiplayer death match game mode. This will hopefully provide a game that’s fun for the user and have a high skill ceiling while also being an interesting programming challenge.

Motivation:

I believe making a game will be a really challenging but rewarding I.T project. The reason for introducing a new mechanic each tutorial level allows us to experiment with different aspects of the game i.e (platforming, gun mechanics, movement) and has the added benefit of being a structured programming challenge. This in turn can allow for us to “follow the fun” (The Games That Designed Themselves, 1:00) in which we can design a better game and explore a fun mechanic that sticks. This will be our first game; we’re not going to be able to make GTA V but will be a good introduction to game design and game programming. The best way to get anywhere in the industry is to “just make stuff”. (How to Become a Game Designer 3:40)

Description:

TUTORIAL:

TUTORIAL: the tutorial levels will start out easy and get progressively harder. They will have clear start and end goal with helpful hints and game tips popping up in the UI. The game mechanics that they will introduce are centred around player movement and gunplay. If time permits there will be an enemy AI to defeat at the end of each level so the player has a chance to learn a little about the combat system before jumping into the deathmatch mode.

Deathmatch:

Deathmatch: The deathmatch mode will consist of 1 map and 2-8 chickens, with the usual rules of the victor being the one with the most kills at the end of the round. How those chickens get those kills will be either a peck mechanic (unarmed), melee weapon, environment death i.e lava or a projectile kill with a variety of guns and a rocket launcher. The guns will spawn randomly and periodically on specific platforms to give the user a dynamic fighting experience.

Gameplay:

Gameplay: The real meat of the game and the core gameplay mechanic that should be focused on to make the game unique are the projectile mechanics. Chickens are not the largest of animals so if they shoot a gun or get shot there should be some consequence or weight behind it. If a chicken shoots a shotgun it should send you flying in the opposite direction. The cool thing about this is that it provides another movement option the player can use to reposition themselves to give them a competitive advantage. The environment should be treacherous with lava pits/ spike traps but also allow for dynamic movement with gravity pads/ grapple hooks locations for an interesting map. A glide mechanic can also be easily implemented to add more dynamic movement; after all they are flightless birds.

Camera:

A third person camera system can be set up using the ‘Cinemachine’ asset in unity. Traditionally it’s very difficult to set up a third person camera system. The outcome of this is that we get a much more responsive camera and flexibility to get the shots we want. Additionally we can add an aim/zoom mechanic with the “create virtual camera function” in the asset.
